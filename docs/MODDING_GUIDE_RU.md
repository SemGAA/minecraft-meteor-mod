# Методичка по созданию больших модов

## 1. С чего реально начинать

Правильный порядок такой:

1. предмет
2. блок
3. крафт/лут
4. простая механика
5. сущность
6. визуал/эффекты
7. большой ивент
8. босс
9. сюжетная цепочка

Если сразу прыгать в “босса с катсценой, сюжетом, диалогами и спецэффектами”, проект почти всегда разваливается.

## 2. Как устроен любой новый объект в моде

Почти всё в Forge-моде делается по одной схеме:

1. зарегистрировать объект в Java
2. добавить ресурсы в `assets`
3. добавить локализацию
4. подключить поведение
5. проверить в игре

Пример:

- новый предмет -> `ModItems.java`
- новый блок -> `ModBlocks.java`
- новая сущность -> `ModEntityTypes.java`
- новая механика мира -> `world/event/*`
- новый пакет -> `network/*`
- новый клиентский визуал -> `client/*`

## 3. Как делать предметы

Минимум для предмета:

1. зарегистрировать в [ModItems.java](/D:/Main/Programing/Minecrat%20mods%20devlop/Forge-Tutorial-1.21.X-1-setup/src/main/java/net/kaupenjoe/tutorialmod/item/ModItems.java)
2. добавить модель в `assets/tutorialmod/models/item/`
3. добавить текстуру в `assets/tutorialmod/textures/item/`
4. добавить имя в `lang/*.json`

## 4. Как делать блоки

Минимум для блока:

1. зарегистрировать блок в [ModBlocks.java](/D:/Main/Programing/Minecrat%20mods%20devlop/Forge-Tutorial-1.21.X-1-setup/src/main/java/net/kaupenjoe/tutorialmod/block/ModBlocks.java)
2. автоматически зарегистрировать `BlockItem`
3. добавить:
   - `blockstates`
   - `models/block`
   - `models/item`
   - `textures/block`
4. если блок “живой”, создать отдельный класс поведения в `block/custom`

## 5. Как делать катсцены правильно

Для красивой катсцены нужен не один файл, а 4 слоя:

1. **серверный сценарий**
   - когда начинается
   - что происходит по тикам
   - кто участвует

2. **сеть**
   - сервер говорит клиенту “начинай катсцену”
   - сервер говорит клиенту “заканчивай катсцену”

3. **камера**
   - лучше отдельная сущность-якорь
   - клиент просто переключает камеру на неё

4. **визуал**
   - сущность события
   - частицы
   - звук
   - рендер

В этом проекте это уже показано на метеорите.

## 6. Как делать боссов

Не начинай с “идеального босса”. Делай слоями:

1. обычная сущность
2. здоровье, урон, модель
3. 1 простая атака
4. 1 спецатака
5. фаза 2
6. арена
7. музыкальный/визуальный сценарий
8. катсцена входа/смерти

Лучше один хороший босс с двумя фазами, чем пять пустых.

## 7. Как делать визуальные эффекты

Есть 3 уровня эффектов:

1. **частицы**
   - быстро
   - дешево
   - хорошо для магии, ауры, заражения

2. **рендер сущности**
   - хорошо для метеоритов, снарядов, призраков, лучей

3. **клиентские экранные эффекты**
   - катсцены
   - тряска
   - засвет
   - затемнение

Для “профессионального” вида лучше сочетать все три, но вводить их по очереди.

## 8. Как делать сюжет и лор

Сюжет в моде держится не на тексте, а на структуре:

1. событие
2. след
3. находка
4. угроза
5. новый враг
6. новая цель
7. развязка

Для твоего направления хорошая цепочка такая:

1. падение метеорита
2. заражённое ядро
3. заражение биома/мира
4. новые мобы
5. культисты или аномалии
6. мини-босс
7. главный босс

## 9. Моды и плагины — в чём разница

Это важно:

- **Forge/NeoForge мод**:
  - новые блоки
  - новые сущности
  - клиентские рендеры
  - катсцены
  - сильные визуальные эффекты

- **Paper/Spigot plugin**:
  - серверная логика
  - команды
  - ивенты
  - RPG-механики
  - мини-игры

Если ты хочешь именно **катсцены, эффекты, кастомных боссов, оружие и атмосферу**, тебе лучше развивать именно мод.

## 10. Как расти дальше по этому проекту

Лучший следующий порядок:

1. сделать второй предмет из метеорита
2. сделать броню или оружие из `alexandrite`
3. сделать заражённого моба
4. сделать мини-босса в кратере
5. добавить структуру падения
6. сделать вторую катсцену — пробуждение босса

## 11. Чеклист перед каждой новой фичей

Перед тем как делать что-то новое, отвечай на 5 вопросов:

1. это серверная логика или клиентская?
2. нужен ли packet?
3. нужен ли отдельный класс?
4. нужен ли свой renderer?
5. как это сохраняется в мире?

Если отвечать на них заранее, код перестаёт расползаться.
